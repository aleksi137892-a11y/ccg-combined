

# Documentation Plan: IIMG Intake Process Guide

I'll create a comprehensive markdown file that documents the entire evidence intake process for the Independent Investigative Mechanism for Georgia (IIMG), walking through each step hierarchically as a user would experience it.

## Document Structure

### 1. **Overview Section**
- Introduction to the IIMG and its mission
- Purpose of the documentation portal
- Key principles: dignity, security, evidentiary standards

### 2. **Entry Point: Language Selection**
- Available languages (Georgian, Russian, Azerbaijani, English)
- Accessibility message and Signal contact for unsupported languages

### 3. **Onboarding Flow (Mental Health & Safety)**
- **Step 1: Documentation with Dignity** - Legal disclaimer and consent
- **Step 2: Medical Check** - Asking if user needs medical attention
- **Step 3: Emotional Support** - Option to speak with someone first
- **Step 4: Affirmation** - "We're here for you" message
- **Step 5: Resources** - Links to MSF, PHR, IRCT, DART organizations
- **Step 6: Security Guidelines** - Safety recommendations and secure contacts
- **Step 7: Ready to Proceed** - Final confirmation before evidence selection

### 4. **Evidence Type Selection**
- Multi-select interface for:
  - "My testimony"
  - "Digital Evidence" (photos, videos)
  - "Physical Materials"
  - "Speak with someone"

### 5. **Tier 1: Testimony Flow**
- Step-by-step testimony collection:
  - What happened?
  - When and where?
  - Who was involved?
  - Contact information (optional)
- Submission with SHA-256 hash verification
- Receipt generation with cryptographic fingerprint

### 6. **Tier 2: Documents Flow**
- Document type selection (Medical, Legal, Official, Financial, Identity, Other)
- Document description
- Date and location information
- Automatic filename generation for chain of custody
- Tresorit encrypted vault upload

### 7. **Tier 3: Digital Evidence (Berkeley Protocol)**
- Overview of cryptographic verification
- Why proper preservation matters
- Save App setup walkthrough:
  - Download app (iOS/Android)
  - Server configuration
  - Enter credentials (server, path, username, password via captcha)
  - Upload process
- Alternative methods: Tresorit, ProtonMail

### 8. **Tier 4: Physical Evidence Flow**
- Safety warnings for chemical contamination
- "Contact us first" guidance
- Preservation guidelines by material type:
  - Clothing & Fabric
  - Liquid Samples
  - Documents & Paper
  - General Storage
- Photograph with Save App
- Secure contact channels

### 9. **Tier 5: Direct Contact (Urgent)**
- Warm, supportive messaging
- Signal and ProtonMail contact options
- Option to return and tell story

### 10. **Completion: Thank You Screen**
- Confirmation of secure documentation
- Security recommendations (clear browser history)
- Secure communication channels for follow-up
- Share options (WhatsApp, copy link)
